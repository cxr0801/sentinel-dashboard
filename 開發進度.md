## 🐛 已修復問題

| 日期 | 問題描述 | 修復方式 |
|------|----------|----------|
| 2026-04-19 | `ModuleNotFoundError: No module named 'requests'` | `pip install requests pandas openpyxl` |
| 2026-04-19 | Apify API 404：Actor ID 含有 `/` 無法直接放入 URL | 改為 `ACTOR_ID.replace("/", "~")` |
| 2026-04-19 | 儀錶板更新後頁面空白：換行符破壞 JS 字串 | 新增 `safe_text()` 函式先清除換行 |
| 2026-04-19 | Apify Webhook Payload 格式不符 GitHub API | 改為 `{"event_type": "apify-done"}` |
| 2026-04-19 | HTML 更新參數 Bug 導致手動觸發時出錯 | 把 `update_dashboard` 中遺漏的 `history` 參數補齊 |
| 2026-04-19 | 歷史趨勢圖每天被重置 | 在 GitHub Actions 腳本加入 `git add summary_history.json` 以儲存歷史 |
| 2026-04-19 | GitHub Actions 因缺失 history 檔引發 git add 報錯 | `git add summary_history.json` 改用 `|| true` 加強防寫防呆 |
| 2026-04-20 | Apify API Token 外洩被官方自動停用爆出 401 | 重新至 Apify 後台生成 Token 並回填至 Github Secrets |
| 2026-04-20 | GitHub Actions 出現 `exit code 1` 靜默崩潰 | 為空 DataFrame 與損壞/未初始化的 JSON 檔加入 `try-except` 防止中斷 |
| 2026-04-20 | `NameError: cannot access free variable 'json'` | 發現 Python 閉包作用域 Bug，刪除 `update_dashboard` 內部多餘的 `import json` |
| 2026-04-20 | GitHub 回傳了 3 天前舊資料，抓不到最新爬蟲結果 | Python `requests` 將 boolean 值發送為 `"True"` 被忽視，強制變更為全小寫 `"true"` 解決 API 排序降冪問題 |
| 2026-04-20 | 儀錶板上的「資料更新時間」比台灣時間慢了 8 小時 | 為所有 `datetime.now()` 設定綁定 `timezone(timedelta(hours=8))` 強制轉為台灣當地時區 |

## 🔄 自動化流程（已上線）

```
每日 Apify 排程爬蟲執行
       ↓ Run succeeded
Apify HTTP Webhook 觸發
       ↓ POST → api.github.com/repos/cxr0801/sentinel-dashboard/dispatches
GitHub Actions 自動執行（約 25 秒）
  1. 安裝 Python 依賴
  2. 執行 auto_daily_analysis.py（從 Github Secrets 注入 APIFY_API_TOKEN）
     -> 下載最新資料並解析
     -> 建立 / 累加 summary_history.json
     -> 修改 sentinel_dashboard.html
  3. git commit & push 將新網頁推回 GitHub
       ↓
GitHub Pages 自動重新部署（約 36 秒）
       ↓
https://cxr0801.github.io/sentinel-dashboard/sentinel_dashboard.html (管理層每日查閱最新資料)
```
